# 온보딩 (비개발자용)

git 을 몰라도 된다. 터미널을 열 필요도 없다. 아래 3단계면 끝.

## 1. 플러그인 설치

- (내부 배포) 관리자가 준 `gitvault-live` 폴더를 vault 의 `.obsidian/plugins/` 에 넣거나 BRAT 로 설치.
- Obsidian 설정 → 커뮤니티 플러그인에서 **gitvault-live** 활성화.

## 2. 저장소 연결 (설정 화면)

플러그인 설정 화면에서 세 가지를 입력한다 (관리자에게 받는다):

1. **저장소 URL** — 예: `https://github.com/owner/vault-repo.git` 또는 사내 GitLab 주소.
2. **사용자명** (선택) — 비우면 호스트 관례로 자동(GitHub `x-access-token`, GitLab `oauth2`, 그 외 `git`).
3. **토큰** — 대상 저장소에 쓰기 권한이 있는 액세스 토큰(GitHub PAT / GitLab access token 등).

**[연결 테스트]** 를 누르면 접속을 확인한다. 특정 플랫폼에 종속되지 않는다 — HTTPS + 토큰을 지원하는 어떤 git 호스트든 동일하게 동작한다.

> 토큰은 이 기기에만 저장된다(플러그인 설정 + `.git/config`). 공용 PC 에서는 사용에 주의.

---

이후로는 **전부 자동**이다.

- 노트를 쓰면 알아서 저장·동기화된다. 상태바에 "동기화됨" 이 뜬다.
- 다른 사람(및 에이전트)이 방금 바꾼 내용은 에디터 본문에 색으로 표시되고, 사이드 패널에서 목록으로도 확인할 수 있다.
- 확정하고 싶을 때 **[저장]** 버튼을 누르면 공식본(main)에 반영된다.

## 문제가 생기면

- 상태바에 "오류" 가 뜨면 잠시 후 자동 재시도한다. 계속 뜨면 토큰 권한/만료를 관리자에게 확인.
- 편집은 오류 중에도 계속 가능하다 — 동기화만 잠시 미뤄질 뿐 데이터는 로컬에 안전하다.
